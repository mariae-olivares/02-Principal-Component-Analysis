Dataset Description and Exploratory Analysis

Dataset Description

El dataset que seleccion para esta tarea fue Blood Transfusion Service Center, por Yeh (2008) y recuperado del UC Irvine Machine Learning Repository. Este dataset corresponde a un problema de clasificacin dentro del rea de negocios y marketing, especficamente en el contexto de campaas del rea de salud.

Los datos fueron recolectados con el propsito de evaluar el modelo RFMTC, una adaptacin del modelo RFM utilizado en marketing, con el objetivo de analizar el historial de donaciones y predecir el comportamiento de los donantes ante campaas de donacin que se realizan cada tres meses.

El dataset contiene informacin de 748 donantes (datos), seleccionados de manera random, y cuenta con 4 variables predictoras:

  • R -> Recency (integer): cantidad de meses desde la ltima donacin.
  • F -> Frequency (integer): nmero total de donaciones realizadas.
  • M -> Monetary (integer): volumen total de sangre donada (en centmetros cbicos).
  • T -> Time (integer): cantidad de meses desde la primera donacin.

La variable a -> objetivo (binaria) representa si el donante particip en la campaa de donacin de sangre de marzo de 2007. Est clasifica de la siguiente manera:

  • 1 = don
  • 0 = no don

Data Cleaning

Para comenzar con el proceso de limpieza de datos, primeramente import las libreras pandas y numpy para poder cargar los datos y realizar los clculos matemticos correspondientes. Posteriormente, sub el dataset y se guard como df. Antes de continuar con el proceso de limpieza, cambi los nombres de las variables (como se explic anteriormente) con el fin de tener una mejor visualizacin y manejo de los datos.

A manera de revisin, se imprimieron los primeros 10 registros del dataset para verificar que la carga se hubiera realizado correctamente.

Ya con este proceso de carga completo, se comenz con la verificacin y limpieza de datos. Primero, se revis la cantidad de valores faltantes utilizando la funcin isnull(). Como resultado, se observ que no existen datos faltantes, lo cual coincide con la descripcin proporcionada por el UC Irvine Machine Learning Repository.

Posteriormente, analic si alguna variable no era relevante para la prediccin de la variable objetivo (a); sin embargo, se concluy que todas las variables aportan informacin importante para el modelo, por lo que no se elimin ninguna. Dado que la base de datos se encontraba bastante limpia, se procedi a realizar un anlisis ms especfico.

En un inicio intent revisar la presencia de outliers; sin embargo, los valores pueden variar considerablemente dependiendo de los hbitos de donacin de cada persona. Mientras que algunos donantes son nuevos, otros pueden donar con mucha frecuencia. Debido a esta variabilidad natural en el comportamiento de los donantes, se decidi no eliminar datos nicamente por presentar valores extremos.

Por otro lado, segn la Organizacin Mundial de la Salud (2022), En la mayora de los pases, el volumen de sangre que se extrae es de 450 mililitros. Con esto en mente, se estableci que debe existir una relacin entre el volumen total de sangre donada (M -> Monetary) y el nmero total de donaciones realizadas (F -> Frequency). Es por ello que cre una nueva columna llamada Vol_esp, donde se multiplica el nmero de donaciones por 450 centmetros cbicos (equivalentes a mililitros), con el fin de estimar el volumen esperado de sangre donada por cada persona.

Antes de comparar los volmenes de sangre, se defini una tolerancia equivalente a 10 sesiones de donacin (4500 mililitros). Posteriormente, mediante un ciclo for, se recorrieron todos los datos calculando la diferencia entre el valor real de volumen de sangre (M) y su valor estimado (Vol_esp). Si la diferencia absoluta superaba la tolerancia establecida, el ndice se almacenaba en una lista llamada inconsistencias. Finalmente, se imprimieron los registros detectados.

Al analizar los resultados, se observ que todas las diferencias eran negativas, es decir, las personas donaron menos volumen del estimado. Por esta razn, se decidi no eliminar dichos datos, ya que es posible que en ciertas ocasiones el volumen donado haya sido menor al estndar. Lo que, en mi opinin, s resultara problemtico sera que el volumen superara considerablemente el lmite recomendado por la OMS. (OMS, 2022)

Se opt por mantener los datos considerando que el tamao de la muestra no es muy grande (menos de 1000 registros). No obstante, tambin sera vlido eliminar estos registros si se considerara necesario.

Principal Component Analysis Procedure

Para esta seccin de la tarea se utilizar el procedimiento de Principal Component Analysis (PCA). De acuerdo con lo visto en clase y con la investigacin realizada, este mtodo se emplea principalmente para reducir la dimensionalidad de conjuntos de datos, transformando las variables originales en un nuevo conjunto de variables no correlacionadas llamadas componentes principales, por ello el nombre del anlisis.(IBM, 2025)

A continuacin se presenta el procedimiento para realizar el PCA:

Data standardisation

Antes de comenzar con el procedimiento de Principal Component Analysis (PCA), es importante aclarar que este mtodo es sensible a la escala de las variables. Si las variables tienen magnitudes muy diferentes, aquellas con mayor varianza pueden dominar el anlisis y afectar la direccin de los componentes principales. Por esta razn, es necesario estandarizar los datos antes de aplicar PCA, para que todas las variables contribuyan de igual manera (IBM, 2025).

Para ello, primero se construy una matriz X que contiene nicamente las variables predictoras del modelo (R, F, M y T) ya que se estar trabajando nicamente con estas. Posteriormente, almacen el nmero total de observaciones en la variable n, la cual se utilizar en prximos clculos .

A partir de la matriz X, calcul las medias (np.mean(X, axis=0)) y desviaciones estndar (np.std(X, axis=0, ddof=1)) de cada variable. Con estos valores se realiz la estandarizacin de los datos utilizando la frmula:

$$X_z = \frac{X - \mu}{\sigma}$$

El resultado es una nueva matriz X_z, cuyos valores tienen media cero (o muy cercana a cero) y desviacin estndar igual a uno. Finalmente, imprim estos valores para comprobar que la estandarizacin se realiz correctamente.

nota: ddof=1 se usa para calcular la desviacin estndar muestral (divide entre n-1)

Covariance matrix

Para continuar con el procedimiento de PCA, calcul la matriz de covarianza a partir de los datos estandarizados X_z utilizando la frmula vista en clase la cual se puede observar en la variable cov.

$$\Sigma_{X_z} = \frac{1}{N-1} X_z^T X_z$$

En los resultados de la matriz de covarianza podemos observar la relacin lineal entre las variables, considerando que estn organizadas en el orden ['R', 'F', 'M', 'T'] tanto en las filas como en las columnas. Esto nos permite identificar fcilmente los valores de interseccin entre cada par de variables, los cuales representan su nivel de correlacin.

En primer lugar, se observa que la diagonal principal est compuesta por valores iguales a 1. Esto confirma que cada variable tiene varianza unitaria, lo cual es tiene sentido con el proceso de estandarizacin realizado anteriormente.

Por otro lado, la covarianza entre Frequency (F) y Monetary (M) es exactamente 1 ( o algo muy cercano a 1). Esto indica una correlacin alta y positiva entre ambas variables. Este resultado tiene lgica, ya que el volumen total de sangre donada depende directamente del nmero total de donaciones realizadas; esto puede llevar a una redundancia en el dataset que seleccion.

Ahora, la variable Time (T) muestra una covarianza positiva considerable con F y M (0.6349). Esto indica que mientras ms tiempo lleva una persona siendo donante, mayor tiende a ser su frecuencia de donaciones y, por lo tanto, el volumen total donado.

En cuanto a Recency (R), se observa una covarianza negativa con F y M (-0.1827). Esto sugiere que mientras mayor sea el tiempo desde la ltima donacin (valores altos de R), menor tiende a ser la frecuencia y el volumen total donado. Esta relacin tambin me resulta lgica, ya que los donantes ms activos suelen haber donado recientemente.

Considerando lo anterior, la matriz nos confirma que existe una fuerte dependencia entre algunas variables, especialmente entre Frequency y Monetary.

Eigenvalues and eigenvectors

Siguiendo con el procedimiento del PCA, se calcularon los eigenvalores y eigenvectores de la matriz de covarianza utilizando la funcin np.linalg.eig(cov).

En PCA, los eigenvectores representan las nuevas direcciones hacia donde se proyectan los datos; es decir, son las direcciones en las que existe mayor variacin ("flechas" que mejor atraviesan la nube de puntos). Por otro lado, los eigenvalores nos dicen cunta varianza hay en cada una de esas direcciones. Un eigenvalor grande significa que los datos estn muy dispersos a lo largo de ese eigenvector, mientras que uno pequeo indica poca dispersin. (IBM, 2025)

Geomtricamente, esto se puede entender como una rotacin del sistema de coordenadas original. En lugar de mantener los ejes originales, el PCA los reorienta hacia nuevas direcciones perpendiculares entre s, alineadas con la mayor dispersin de los datos. De esta forma, la mayor parte de la informacin (varianza) se queda en los primeros componentes principales. (IBM, 2025)

Antes de continuar con el procedimiento del PCA, es necesario ordenar los eigenvalores y eigenvectores de mayor a menor. Esto se hace porque el primer componente principal debe tener la mxima varianza. Visualmente, es como poner primero la "flecha" ms larga (la direccin ms importante).

Para esto, utilic la funcin np.argsort(eigvals) para obtener los ndices que ordenan los eigenvalores de menor a mayor. Luego, con [::-1] se invierte ese orden para que queden de mayor a menor y se guarda el orden en idx. Posteriormente, se reordenan tanto los eigenvalores como los eigenvectores usando esos ndices. Esto es importante porque cada eigenvector debe mantenerse alineado con su eigenvalor correspondiente.

Transformation matrix

En esta parte del cdigo primero se define la matriz de transformacin (que corresponde a los eigenvectores ordenados); cada columna de esta matriz representa una nueva direccin en el espacio, es decir, un componente principal. Posteriormente, se utiliza la formula vista en clase para obtener los datos transformados en trminos de las componentes principales (X_pca = X_z.dot(F)):

$$X_{pca} = X_z \cdot F$$

Para verificar que todo se realiz de manera correcta, se imprimen los primeros 5 datos transformados.

Explained variance

En esta parte del proceso, se calcula el porcentaje de varianza explicada por cada componente principal mediante exp_V = eigvals / np.sum(eigvals), que obtuve de la frmula vista en clase. Este paso nos permite identificar cuntos componentes son necesarios para conservar la mayor parte de la informacin del dataset.

$$V_{\lambda_j} = \frac{\lambda_j}{\sum_{i=1}^M \lambda_i}$$

Los resultados muestran que el primer componente principal explica aproximadamente el 63.5% de la varianza total, mientras que el segundo componente explica un 27.5%. Es decir, que los dos primeros componentes acumulan un 91% de la varianza, esto nos dice que la mayor parte de la informacin en los datos originales puede representarse adecuadamente en un espacio 2D. El tercer componente solo aporta un 8.9%, y el cuarto componente tiene un valor despreciable, lo que sugiere que puede ignorarse sin perder informacin.

Conclusiones

En conclusin, esta tarea en lo personal represent un reto para m porque me llev a comprender realmente el procedimiento detrs del PCA y no solo aplicarlo como una funcin. Aunque el dataset seleccionado cuenta nicamente con cuatro variables predictoras, el anlisis me permiti evaluar la efectividad del PCA en el contexto especfico. Si bien no se logr una reduccin drstica en el nmero de variables, el PCA s fue efectivo al concentrar aproximadamente el 91% de la varianza en los primeros dos componentes principales.

Uno de mis principales aprendizajes fue entender que la reduccin de dimensionalidad no solamente es eliminar variables, sino organizar la informacin para capturar la mayor variabilidad posible en menos componentes. En este caso, el PCA ayud especialmente a manejar la alta correlacin entre Frequency y Monetary, ya que ambas variables aportaban informacin muy similar.

En cuanto al espacio reducido para modelos como regresin o clasificacin, considero que el uso de los primeros dos o tres componentes principales puede ser adecuado. Reducir dimensiones disminuye el costo computacional, y puede ayudar a evitar sobreajuste, mejorando la estabilidad del modelo. Para un problema de clasificacin como este, donde la variable objetivo es binaria, trabajar con dos componentes podra ser suficiente, ya que conservan la mayor parte de la varianza. Sin embargo, incluir el tercer componente (explica alrededor del 8.9%) podra aportar informacin adicional relevante dependiendo del modelo.

En general, aunque el dataset no tena muchas variables predictorias, el PCA me permiti comprender mejor la estructura interna de los datos y la relacin entre variables. Ms que reducir muchas variables, el verdadero aporte fue entender cmo se redistribuye la informacin en nuevas direcciones y cmo esto puede beneficiar a mis futuros modelos de regresin o clasificacin.
